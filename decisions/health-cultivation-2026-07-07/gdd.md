# 「道途」健康修仙模拟器 — Game Design Doc (GDD)
*Version 0.1 · 2026-07-07 · Author: Hermes Agent*

---

## 一、产品一句话定位

> **「道途」** — 把"专注、运动、睡眠、冥想"等真实健康行为,包装成"修仙修炼",用东方修仙文化叙事 + 数字水墨 UI,把现代人最讨厌的"健康打卡"变成"修仙旅程"。
> **核心心智**: 你不是在"打卡",你是在"精进"。

---

## 二、核心循环(MUST)

```
┌──────────────────────────────────────────────────────┐
│  做健康行为 → 记录修炼 → +修为 → 累积 → 突破境界     │
│      ↑                                   ↓           │
│      └────────── 仪式感反馈 ←─────────────┘           │
└──────────────────────────────────────────────────────┘
```

**最小闭环验证**(D3-D5 必须跑通):
1. 用户做 20 分钟专注 → 标记"今日悟道" → +0.5 修为 → 看到进度条走了一格
2. 用户做 30 分钟运动 → 标记"今日锻体" → +0.3 修为 → 看到体力计数变化
3. 用户睡满 7 小时(隔夜)→ 标记"昨夜静修" → +1.5 修为 → 看到静修 timer 计数
4. 累计修为达到境界阈值 → **仪式感动画** + 进阶

---

## 三、修炼分类(三大体系,五小类)

| 类型 | 用户场景 | 修为奖励 | 心率上限 | 解锁境界 |
|---|---|---|---|---|
| **悟道** (focus) | 学习/深度工作 | +0.5/20min | 必须 ≥ 20min 持续 | 筑基即解锁 |
| **打坐** (短冥想) | 番茄钟/小憩 | +0.2/5min | ≥ 5min | 筑基即解锁 |
| **锻体** (exercise) | 跑步/健身/搬砖 | +0.3/30min | ≥ 30min | 筑基即解锁 |
| **搬砖** (体力活) | 做家务/购物 | +0.15/30min | ≥ 30min | 筑基即解锁 |
| **行气** (breath) | 5分钟呼吸练习 | +0.1/5min | ≥ 5min | 筑基即解锁 |
| **静修** (sleep) | 睡眠 ≥ 7小时 | +1.5/夜 | 必须 6-10小时 | 筑基即解锁 |
| **历练** (外出) | 远足/旅行 | +0.5/2h | ≥ 2h 离开常规地 | 筑基即解锁 |

**核心规则**:
- 用户可事后补录(今早忘了,晚上补打),**永远不锁死**
- 时间必须真实,app 不自动计时,信任用户(防作弊)
- 不做"中断判定" — 用户说断就是断了,只奖励已完成部分

---

## 四、六大境界(晋升路线)

| 境界 | 累计修为 | 实际天数(混合修炼) | 解锁内容 |
|---|---|---|---|
| **炼气期** | 0 | 0 | 基础修炼 (悟道/行气) |
| **筑基** ✅ | 50 | 7-10 天 | 全类修炼 + 第一卷功法 |
| **金丹** | 200 | 30 天 | 渡劫仪式(可选) + 第二卷功法 |
| **元婴** | 600 | 90 天 | 天人感应(自定义仪式) + 第三卷功法 |
| **化神** | 1500 | 180 天 | 一键分享成就 + 第四卷功法 |
| **渡劫** 🎉 | 3000 | 365 天 | **飞升仪式**(全屏动画 + 文言宣告) |

**仪式感设计**(核心差异化):
- 境界突破时:**1.5 秒暗场** → 全屏水墨画 → 慢速旁白("恭喜道友,筑基已成...")+ 金粉特效 → 用户点击"继续精进"返回主页
- 飞升时:**5 秒仪式动画**,可截屏分享(无社交,纯个人纪念)
- 仪式旁白音频可后期用 edge-tts 生成,先静音

---

## 五、6 个必须避的坑(来自竞品/视频 UP 主踩过的坑)

1. ❌ **不做"硬卡死时间"规则** —— 用户睡过了/忘了,要能补录
2. ❌ **不做"任务断检"机制** —— 一直后台监测心率/键盘会很烦
3. ❌ **不做"强迫每日打卡"** —— 哪天没做,修为不增不减,不要"今日为破功"的红字警告
4. ❌ **不做"完整 RPG 系统"** —— 不开天赋树/装备/技能,只保留"修炼 × 修为 × 境界"三件套
5. ❌ **不搞社交** —— 1.0 完全不需要,后期再说(国内备案风险)
6. ❌ **不做支付/会员** —— 1.0 纯免费,任何商业化都推迟到 D60 后

---

## 六、6 个 MVP 必做功能

1. **修炼记录 UI**:5 个按钮(悟道/锻体/行气/静修/其它)+ 时长输入 + "完成修炼" 提交
2. **修为/境界进度条**:首页顶部,大字号,看一眼就知道今天精进了多少
3. **境界进度分布**:底部 Tab,显示今日/本周/本月/累计修为(简单 4 个数字 + 4 个折线图)
4. **突破仪式**:修为过阈值时自动触发 1.5-5 秒动画
5. **历史记录**:listview 看到所有过往修炼(可搜索/筛选)
6. **设置**:个人信息(道号、性别、生日 — 仅用于干支推算)、数据导出

## 七、视觉规范 (水墨沉兰)

| 元素 | 规格 |
|---|---|
| 主色 | 墨绿 `#2c4a3e` / 米白 `#f5f0e6` |
| 辅色 | 金粉 `#c9a961` / 远山青 `#7d9d8c` |
| 字体 | 思源宋体 (免费) — 标题 + 衬线 |
| 标题字号 | 28-36px,字重 600 |
| 正文字号 | 16px |
| 边角 | 大圆角 (12-20px) |
| 装饰元素 | 远山轮廓 SVG / 流云 / 卷轴卡片 |
| 动画时长 | 0.3-0.8s 缓出 (cubic-bezier 0.4, 0, 0.2, 1) |
| **禁用** | 紫色/蓝色糖果色/任何 emoji |

**文案语调**:
- ❌ "今日你修炼了 30 分钟,获得 0.5 修为"
- ✅ "道友今日精进,悟道 30 分钟,获修为 +0.5"
- ❌ "点击此处开始练习"
- ✅ "道友,可要开始今日悟道?"

## 八、技术选型

| 层 | 选择 | 理由 |
|---|---|---|
| 微信小程序 | **Taro 3** | 一次开发,小程序/H5 双发 |
| 后端 | **Supabase** | Postgres + Auth + 实时订阅,**免备案**,海外访问稳定 |
| 本地存储 | **Taro Storage** | 离线优先,网络不好也能记 |
| 动画 | **Lottie** + CSS Transition | 包小 + 高质量 |
| 推送 | 微信订阅消息 | 仅在用户主动订阅后提醒 |
| CI | GitHub Actions + 微信开发者工具 | 自动构建小程序预览 |

> 国内微信小程序备案需要 7-14 天 + 营业执照,但 MVP 阶段可以先跑 **H5 + 微信内嵌** 走"轻量发布"

## 十、用户成功指标(北极星)

| 指标 | 目标(D30) |
|---|---|
| 当日留存(D1) | ≥ 60% |
| 周留存(D7) | ≥ 30% |
| 月留存(D30) | ≥ 15% |
| 30 天累计修炼次数 | ≥ 50 次/活跃用户 |
| 30 天突破境界比例 | ≥ 50%(炼气→筑基) |

**核心问题**:不是"DAU",而是"30天后还在用的比例"。

---

## 十一、显式不做清单 (Anti-Goals)

为了保护"轻量专注"的产品定位,以下功能**永久延后**:

- ❌ 多端同步 (1.0 只用本地)
- ❌ 社交(关注/动态/比较)
- ❌ AI 教练 / Chat
- ❌ 商品/虚拟道具/会员
- ❌ 用户生成内容 (UGC) 通道
- ❌ 强迫通知 (任意 ads)
- ❌ 跨修炼法(八段锦/养生)内容
- ❌ 强制实名/真实身份验证
