## 一、cognee 是什么:重新定义 AI Agent 记忆层

**官方一句话定位**: "The open-source AI memory platform for agents" — 让 AI Agent 拥有跨会话的持久长期记忆,在自托管的知识图谱上持续构建语义网络。

**与传统方案的差异**:

| 维度 | 传统方案 | cognee |
|---|---|---|
| 本质 | 向量数据库 + 检索增强 | 向量+图+认知科学的统一认知层 |
| 关系理解 | 仅语义相似度,不懂结构 | 自动推断实体/关系/从属/因果 |
| 反馈学习 | 静态,不会进化 | `improve` API 根据反馈更新边权重 |
| 接口复杂度 | 多套 SDK 各管一段 | 4 个 API (remember/recall/forget/improve) |
| 多租户 | 应用层隔离 | 存储层物理隔离 |

## 三、ECL Pipeline:核心数据处理机制

cognee 的数据处理核心是三阶段流水线:

```
Extract (提取)  →  Cognify (认知化)  →  Load (加载)
   38+ 数据源       实体+关系+本体        三层混合存储
   自动分块        自动本体推断           SQLite/LanceDB/Kuzu
```

### 3.1 Extract 阶段
- 支持 **38+ 种数据源**:不只文本/PDF,还包括 Slack 对话、音频、图片、结构化数据库表
- 视频提到的"38 种数据源"经 README 验证为多模态摄取能力

### 3.2 Cognify 阶段(真正的魔法)
- **不是简单切块嵌入**,而是从文本中提取:
  - 实体(NER)
  - 实体间关系(RE)
  - 向量表示
  - **本体结构**(自动从数据推断 schema,无需手动定义)
- **认知科学原理**:受人类记忆模型启发 — 编码→存储→检索→遗忘四个认知循环都有对应机制

### 3.3 Load 阶段(三层混合存储)

| 层 | 存储 | 用途 | 替代方案 |
|---|---|---|---|
| **关系层** | SQLite | 元数据精确查询(作者/创建时间) | PostgreSQL |
| **向量层** | LanceDB | 语义相似度检索 | Qdrant, Weaviate, Milvus |
| **图层** | Kuzu (本地默认) | 实体关系推理 | Neo4j, Amazon Neptune |

> 关键:**同一 API 在三层之间自动路由,对用户透明**

## 四、四大 API:把复杂知识工程浓缩成四行代码

```python
import cognee, asyncio

async def main():
    # 1. remember - 永久存入知识图谱
    await cognee.remember("Cognee turns documents into AI memory.")

    # 会话级快速缓存,后台异步同步到图谱
    await cognee.remember("User prefers detailed explanations.", session_id="chat_1")

    # 2. recall - 智能查询,自动选最优检索策略
    results = await cognee.recall("What does Cognee do?")
    for r in results: print(r)

    # 3. forget - 精确删除(隐私合规场景关键)
    await cognee.forget(dataset="main_dataset")

    # 4. improve - 用户反馈更新图边权重(让记忆会进化)
    # 视频里的"InPro"实际 API 名是 improve

asyncio.run(main())
```

### 4.1 `remember`
- 接收任意数据(文本/对话/文件)
- 跑完 ECL Pipeline
- 永久存入知识图谱
- **支持会话级记忆**:临时信息先存快速缓存,后台异步同步

### 4.2 `recall`
- **自动路由**:不需要关心底层用向量检索还是图遍历
- **混合模式准确率 92.5%**(纯向量方案通常 70-80%)
- 会话优先,未命中穿透到图谱

### 4.3 `forget`
- 精确删除某个数据集/某条对话记录
- 满足 GDPR / HIPAA 等隐私合规要求

### 4.4 `improve`(视频称"InPro")
- 最具想象力的 API
- 用户对 Agent 回答评分 → 反馈更新图谱边权重
- **记忆在使用中进化,而不是静态**

## 五、核心架构创新:三层混合存储

### 5.1 为什么需要三层?

大部分记忆系统只有一层:要么是向量数据库,要么是图数据库。
- **只用向量**:能找语义相近的文本块,但不知道"库克是苹果的 CEO"
- **只用图**:精确但不能处理模糊查询
- **三层融合**:精确查询 + 语义相似 + 关系推理同时具备

### 5.2 实际案例对比
- 视频原话:"它能找到苹果公司和蒂姆·库克的相关段落,但不知道库克是苹果的 CEO"
- cognee 的解法:向量层找出相关文本块 → 图层补全实体关系 → 用户得到既相关又结构化的答案

## 十一、铁律 / 反常识(视频重点强调)

1. **不是又一层向量数据库的包装** — 向量层只是 cognee 的三分之一,真正壁垒是图谱推理 + 认知科学本体
2. **图谱是核心,不是附件** — mem0 把图谱锁在 $249/月付费版,cognee 免费开放,这是产品哲学差异
3. **记忆要会进化** — `improve` API 是杀手锏,用户反馈直接更新图边权重,而不是只训练模型
4. **多租户必须在存储层做** — 应用层隔离在合规场景不够,cognee 直接在 SQLite/LanceDB/Kuzu 物理隔离
5. **AI 记忆是学术问题** — 团队发了 arXiv 论文(2505.24478)讲 KG-LLM 接口优化,有理论支撑
